# 네임스페이스 마이그레이션 가이드 (Namespace Migration Guide)

[English](../namespace-migration.md)

이 문서는 네임스페이스 정책 변경에 따른 운영상의 영향과 기존 데이터를 마이그레이션하는 방법을 설명합니다.

## 변경 사항 요약 (Summary of Changes)

CLI와 MCP 도구 간의 일관성을 높이기 위해 기본 네임스페이스 처리 방식이 통일되었습니다.

### 명시적인 "default" 네임스페이스
기본 네임스페이스는 이제 명시적으로 `"default"` 문자열이 됩니다. 이전에는 네임스페이스가 설정되지 않은 경우 데이터베이스에서 빈 문자열(`""`)로 취급되는 경우가 많았으며, 이로 인해 빈 문자열을 가정하는 도구와 "default"를 가정하는 도구 간에 일관되지 않은 동작이 발생했습니다.

### 암시적인 전체 데이터베이스 읽기 제거
네임스페이스를 지정하지 않았을 때 모든 네임스페이스를 스캔하던 암시적인 동작이 다음 경로에서 제거되었습니다:
- `status`
- `docs`
- `index`
- `lint`
- `eval`
- 모든 MCP 관련 도구 경로

기본(default) 데이터가 아닌 다른 데이터를 조작하려면 이제 반드시 네임스페이스를 명시해야 합니다.

### CLI 동작
전역 `--namespace` 플래그의 기본값이 `"default"`로 설정되었습니다. 네임스페이스를 비워두거나 설정하지 않아도 더 이상 예전의 빈 문자열 동작이 트리거되지 않습니다.

## 마이그레이션 및 백필 (Migration and Backfill)

### 자동 백필 (Automatic Backfill)
시스템 시작 시, `namespace = ''`인 기존 레코드를 `namespace = 'default'`로 자동으로 백필(backfill)합니다. 이를 통해 기존 데이터가 새로운 기본 정책 하에서도 계속 접근 가능하도록 보장합니다.

### 충돌 처리 (Collision Handling)
백필 작업 중 충돌(예: 동일한 엔티티에 대해 `''`와 `'default'` 네임스페이스가 모두 존재함)이 감지되면 마이그레이션은 즉시 실패 처리됩니다. 데이터 오염을 방지하기 위해 시스템은 이러한 레코드를 암시적으로 병합하지 않습니다. 만약 이 문제가 발생하면 시스템을 다시 시작하기 전에 데이터베이스에서 수동으로 중복을 해결해야 합니다.
